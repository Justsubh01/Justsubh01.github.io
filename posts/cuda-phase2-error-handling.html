CUDA Mastery Phase 2.3 — Error Handling in CUDA: Never Ignore cudaSuccess

Why CUDA errors are silent by default and how that destroys debugging sessions, how to write bulletproof error-checking macros, and the complete toolkit for catching every type of CUDA error before it bites you in production.

I've watched engineers spend days debugging a subtle GPU bug that a single cudaGetLastError() call would have caught in minutes. CUDA's asynchronous execution model means errors can appear far from their source. If you don't check every API call, you're flying blind.

Why CUDA Errors Are Silent

  // DANGEROUS — no error checking:
  cudaMalloc(&d_ptr, bytes);        // returns cudaErrorMemoryAllocation?
  cudaMemcpy(d_ptr, h_ptr, bytes,   // using invalid d_ptr — undefined behavior!
             cudaMemcpyHostToDevice);
  kernel<<<grid, block>>>(d_ptr);  // kernel gets garbage input
  cudaMemcpy(h_result, d_out, ...); // copies garbage result

  // You see wrong numbers. You have no idea why.
  // CUDA didn't crash — it just silently failed upstream.

Almost all CUDA runtime API functions return a cudaError_t. If you don't capture and check that return value, errors are silently ignored. This is C — no exceptions, no automatic error propagation.

The Standard CUDA_CHECK Macro

#define CUDA_CHECK(call) do {                                         \
    cudaError_t _err = (call);                                        \
    if (_err != cudaSuccess) {                                        \
        fprintf(stderr,                                               \
                "CUDA error at %s:%d — %s\n",                        \
                __FILE__, __LINE__,                                   \
                cudaGetErrorString(_err));                            \
        exit(EXIT_FAILURE);                                           \
    }                                                                 \
} while (0)

// Usage — wrap every CUDA call:
CUDA_CHECK(cudaMalloc(&d_a, bytes));
CUDA_CHECK(cudaMemcpy(d_a, h_a, bytes, cudaMemcpyHostToDevice));
CUDA_CHECK(cudaDeviceSynchronize());
CUDA_CHECK(cudaFree(d_a));

The do { ... } while(0) wrapping is important. Without it, if (condition) CUDA_CHECK(call); else something(); breaks because the macro expands to two statements and the else binds to the wrong if.

Checking Kernel Launches

The <<<...>>> launch syntax returns void — you cannot assign its return value. Use cudaGetLastError() immediately after: 

// Kernel launch:
myKernel<<<grid, block>>>(args);

// Check for launch configuration errors (synchronous):
cudaError_t launchErr = cudaGetLastError();
if (launchErr != cudaSuccess) {
    fprintf(stderr, "Kernel launch failed: %s\n",
            cudaGetErrorString(launchErr));
    exit(EXIT_FAILURE);
}

// Check for kernel EXECUTION errors (asynchronous — must synchronize first):
cudaError_t execErr = cudaDeviceSynchronize();
if (execErr != cudaSuccess) {
    fprintf(stderr, "Kernel execution failed: %s\n",
            cudaGetErrorString(execErr));
    exit(EXIT_FAILURE);
}

// Or combine:
#define CUDA_KERNEL_CHECK() do {                                      \
    CUDA_CHECK(cudaGetLastError());                                   \
    CUDA_CHECK(cudaDeviceSynchronize());                              \
} while(0)

myKernel<<<grid, block>>>(args);
CUDA_KERNEL_CHECK();

Synchronous vs Asynchronous Errors

  Two classes of CUDA errors:

  SYNCHRONOUS (detected immediately):
  ─────────────────────────────────────────────────────────
  • Invalid launch parameters: negative block size, too many threads
  • cudaMalloc with size 0 (on some versions)
  • Calling CUDA API before cudaInit
  → Returned directly by the API function call

  Example:
  cudaError_t err = cudaMalloc(&ptr, -1);  // invalid size
  // err == cudaErrorInvalidValue immediately

  ASYNCHRONOUS (detected after kernel runs):
  ─────────────────────────────────────────────────────────
  • Out-of-bounds memory access inside kernel
  • Stack overflow in device code
  • Division by zero (on some hardware)
  → Returned by next synchronizing call (Memcpy, Synchronize)
  → cudaGetLastError() sticky: error persists until cleared

  Danger: an async error from kernel K may appear as an error
  in cudaMemcpy called much later!

  Solution: cudaDeviceSynchronize() right after kernels in debug builds.

Common CUDA Errors and Their Meanings

  cudaSuccess (0)
    Everything is fine.

  cudaErrorMemoryAllocation
    cudaMalloc failed — not enough GPU memory.
    Fix: reduce allocation size, free unused memory first,
         check nvidia-smi for memory usage.

  cudaErrorInvalidValue
    Wrong argument (NULL pointer, zero size, invalid enum).
    Fix: check all arguments to the failing call.

  cudaErrorInvalidDevicePointer
    Passing a HOST pointer where a DEVICE pointer is expected.
    Classic bug: accidentally using h_ptr instead of d_ptr.

  cudaErrorLaunchFailure
    Kernel crashed — illegal memory access, stack overflow.
    Use compute-sanitizer for details.

  cudaErrorNoDevice
    No CUDA GPU found. Check nvidia-smi.

  cudaErrorCudartUnloading
    Using CUDA after the runtime has shut down (after main() returns).
    Fix: call cudaDeviceReset() before exit, or restructure code.

  cudaErrorIllegalAddress
    Out-of-bounds memory access in kernel. The most common serious error.
    Fix: check index calculations and bounds guards.

compute-sanitizer — The Valgrind for CUDA

  # Run memcheck (catches illegal memory accesses):
  compute-sanitizer --tool memcheck ./my_cuda_program

  # Run racecheck (catches shared memory race conditions):
  compute-sanitizer --tool racecheck ./my_cuda_program

  # Run synccheck (catches __syncthreads() issues):
  compute-sanitizer --tool synccheck ./my_cuda_program

  # Run initcheck (catches uninitialized device memory reads):
  compute-sanitizer --tool initcheck ./my_cuda_program

  # Run all checks:
  compute-sanitizer ./my_cuda_program

Debug Build Flags

  # Debug build: adds line number info to device code, disables optimizations
  nvcc -g -G -O0 -o my_program my_program.cu
  #    -g  = host debug symbols (for gdb)
  #    -G  = device debug symbols (for cuda-gdb, compute-sanitizer)
  #    -O0 = no optimization (makes stepping in debugger sane)

  # WARNING: -G drastically slows down GPU code (10-100x)
  # Only use for debugging, never for benchmarking!

  # For production:
  nvcc -O3 -use_fast_math -o my_program my_program.cu

A Production-Ready Error Handling Template

#include <cuda_runtime.h>
#include <stdio.h>
#include <stdlib.h>

// ─────────────────────────────────────────────────────────────────
// Error checking macros
// ─────────────────────────────────────────────────────────────────

#define CUDA_CHECK(call) do {                                         \
    cudaError_t _e = (call);                                          \
    if (_e != cudaSuccess) {                                          \
        fprintf(stderr, "[CUDA ERROR] %s:%d\n  %s\n",                \
                __FILE__, __LINE__, cudaGetErrorString(_e));          \
        exit(EXIT_FAILURE);                                           \
    }                                                                 \
} while(0)

// Use this after every kernel launch:
#define CUDA_KERNEL_CHECK() do {                                      \
    cudaError_t _e = cudaGetLastError();                              \
    if (_e != cudaSuccess) {                                          \
        fprintf(stderr, "[KERNEL LAUNCH ERROR] %s:%d\n  %s\n",       \
                __FILE__, __LINE__, cudaGetErrorString(_e));          \
        exit(EXIT_FAILURE);                                           \
    }                                                                 \
    _e = cudaDeviceSynchronize();                                     \
    if (_e != cudaSuccess) {                                          \
        fprintf(stderr, "[KERNEL EXEC ERROR] %s:%d\n  %s\n",         \
                __FILE__, __LINE__, cudaGetErrorString(_e));          \
        exit(EXIT_FAILURE);                                           \
    }                                                                 \
} while(0)

// ─────────────────────────────────────────────────────────────────
// Safe CUDA allocation helpers
// ─────────────────────────────────────────────────────────────────

static inline float* cuda_alloc_float(int n) {
    float *ptr = NULL;
    CUDA_CHECK(cudaMalloc(&ptr, n * sizeof(float)));
    return ptr;
}

static inline void cuda_upload(void *d, const void *h, size_t bytes) {
    CUDA_CHECK(cudaMemcpy(d, h, bytes, cudaMemcpyHostToDevice));
}

static inline void cuda_download(void *h, const void *d, size_t bytes) {
    CUDA_CHECK(cudaMemcpy(h, d, bytes, cudaMemcpyDeviceToHost));
}

// ─────────────────────────────────────────────────────────────────
// Example usage
// ─────────────────────────────────────────────────────────────────
int main() {
    const int N = 1 << 20;

    float *d_a = cuda_alloc_float(N);
    float *d_b = cuda_alloc_float(N);
    float *d_c = cuda_alloc_float(N);

    // ... initialize h_a, h_b
    cuda_upload(d_a, h_a, N * sizeof(float));
    cuda_upload(d_b, h_b, N * sizeof(float));

    vectorAdd<<<(N+255)/256, 256>>>(d_a, d_b, d_c, N);
    CUDA_KERNEL_CHECK();   // catches both launch and execution errors

    cuda_download(h_c, d_c, N * sizeof(float));

    CUDA_CHECK(cudaFree(d_a));
    CUDA_CHECK(cudaFree(d_b));
    CUDA_CHECK(cudaFree(d_c));
    return 0;
}

Error Handling in Production vs Development

  Development:
  ─────────────────────────────────────────────
  • Check EVERY call — CUDA_CHECK on everything
  • Add CUDA_KERNEL_CHECK() after every kernel launch
  • Build with -g -G for compute-sanitizer
  • Run compute-sanitizer regularly
  • Test with invalid inputs intentionally

  Production:
  ─────────────────────────────────────────────
  • Keep CUDA_CHECK on allocations (OOM is real in production)
  • Remove CUDA_KERNEL_CHECK() synchronize (performance cost)
  • Use async error checking only at pipeline boundaries
  • Log errors, don't crash (return error code, retry, fallback)
  • Build with -O3 -use_fast_math (no -G)

Takeaways

  • Every CUDA API call returns cudaError_t. Ignoring it means silent failures.
  • Kernel launches return void — use cudaGetLastError() immediately after.
  • Asynchronous errors appear at the next synchronizing call — synchronize early in debug builds.
  • compute-sanitizer --tool memcheck catches illegal memory accesses in kernels.
  • Build with -g -G for debugging; never benchmark with these flags.
  • The CUDA_CHECK macro pattern is the industry standard — use it everywhere.
