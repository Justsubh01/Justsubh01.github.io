CUDA Streams — Overlapping Copy and Compute for Free Speedup

A stream is a sequence of CUDA operations that execute in order. Multiple streams run concurrently. By pipelining host-to-device copies, kernel execution, and device-to-host copies across streams, you can overlap all three — turning 3 sequential phases into a pipeline that runs at the speed of the slowest phase.

Default stream (top): H→D copy, kernel, D→H copy run strictly sequentially. Multi-stream pipeline (bottom): while chunk N's kernel runs, copy chunk N+1 from host and copy chunk N-1 results back — all three engines run simultaneously. Total time shrinks to the compute-bound limit.

The Default Stream and Why It Serializes

Every CUDA operation you perform without specifying a stream goes into the default stream (stream 0). Operations in the default stream execute in order, and the default stream also blocks all other streams — it is a global synchronization point. 

// Default stream: everything serialized, PCIe and compute engines never overlap
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);  // blocks until done
kernel<<<grid, block>>>(d_A, d_B, N);               // waits for copy to finish
cudaMemcpy(h_B, d_B, bytes, cudaMemcpyDeviceToHost);  // waits for kernel to finish

// Timeline:
// [=====H2D=====][====KERNEL====][=====D2H=====]
// Total: T_copy + T_kernel + T_copy = 3 phases serial
What can overlap on an A100: (1) One H→D PCIe copy engine, (2) One D→H PCIe copy engine, (3) Multiple compute kernels. All three can run simultaneously on separate hardware engines. Streams let you exploit this.

Creating and Using Streams

// Create streams
cudaStream_t stream1, stream2, stream3;
cudaStreamCreate(&stream1);
cudaStreamCreate(&stream2);
cudaStreamCreate(&stream3);

// Assign operations to streams (4th arg to cudaMemcpyAsync, or <<<g,b,smem,stream>>>)
cudaMemcpyAsync(d_A, h_A, bytes, cudaMemcpyHostToDevice, stream1);
kernel<<<grid, block, 0, stream1>>>(d_A, d_result, N);
cudaMemcpyAsync(h_result, d_result, bytes, cudaMemcpyDeviceToHost, stream1);

// Different operations in stream2 run concurrently with stream1
cudaMemcpyAsync(d_B, h_B, bytes, cudaMemcpyHostToDevice, stream2);
kernel2<<<grid, block, 0, stream2>>>(d_B, d_result2, N);

// Synchronize all streams
cudaDeviceSynchronize();

// Or synchronize one stream:
cudaStreamSynchronize(stream1);

// Cleanup
cudaStreamDestroy(stream1);
cudaStreamDestroy(stream2);
cudaStreamDestroy(stream3);
cudaMemcpyAsync requires pinned (page-locked) memory on the host side. Regular malloc() memory will silently fall back to synchronous copy. Use cudaMallocHost() or cudaHostAlloc() for host memory used in async copies.

Pinned Memory — Required for Async Copies

Normal malloc memory is pageable — the OS can swap it to disk. The GPU DMA engine cannot access pageable memory directly. For async copies, CUDA needs pinned (page-locked) memory: guaranteed to stay in physical RAM. 

// Allocate pinned memory:
float* h_data;
cudaMallocHost(&h_data, N * sizeof(float));  // pinned, page-locked
// OR with flags:
cudaHostAlloc(&h_data, N * sizeof(float), cudaHostAllocDefault);

// Initialize
for (int i = 0; i < N; i++) h_data[i] = i * 0.1f;

// Now async copies work truly asynchronously:
cudaMemcpyAsync(d_data, h_data, N * sizeof(float),
                cudaMemcpyHostToDevice, stream1);
// This returns IMMEDIATELY on the CPU while DMA runs in background

// Free:
cudaFreeHost(h_data);

Pinned memory bandwidth comparison:

# cudaMemcpy performance (A100, 1 GB transfer): Pageable host → Device: 6.1 GB/s (PCIe 4.0 limit: 31.5 GB/s) Pinned host → Device: 24.3 GB/s (4x faster, near PCIe limit) Device → Pinned host: 21.8 GB/s Device → Pageable host: 5.9 GB/s # Pinned memory enables overlap, pageable does NOT
Pinned memory is a limited resource. Pinning too much memory degrades system performance for all other processes (less physical RAM available for paging). A practical rule: pin no more than 25% of system RAM. For 64 GB system: <16 GB pinned.

The Double-Buffer Pipeline Pattern

The classic streams use case: processing a large array in chunks, overlapping the copy of chunk N+1 with the computation of chunk N. 

void processWithStreams(float* h_input, float* h_output, int N, int chunkSize) {
    int nStreams = 2;  // double-buffer
    int nChunks  = (N + chunkSize - 1) / chunkSize;

    // Allocate pinned host memory
    float* h_in[2];
    float* h_out[2];
    cudaMallocHost(&h_in[0],  chunkSize * sizeof(float));
    cudaMallocHost(&h_in[1],  chunkSize * sizeof(float));
    cudaMallocHost(&h_out[0], chunkSize * sizeof(float));
    cudaMallocHost(&h_out[1], chunkSize * sizeof(float));

    // Allocate device memory
    float* d_in[2];
    float* d_out[2];
    cudaMalloc(&d_in[0],  chunkSize * sizeof(float));
    cudaMalloc(&d_in[1],  chunkSize * sizeof(float));
    cudaMalloc(&d_out[0], chunkSize * sizeof(float));
    cudaMalloc(&d_out[1], chunkSize * sizeof(float));

    cudaStream_t streams[2];
    cudaStreamCreate(&streams[0]);
    cudaStreamCreate(&streams[1]);

    dim3 block(256);
    dim3 grid((chunkSize + 255) / 256);

    for (int chunk = 0; chunk < nChunks; chunk++) {
        int s = chunk % 2;  // alternate between 0 and 1
        int offset = chunk * chunkSize;
        int thisChunk = min(chunkSize, N - offset);

        // Copy input chunk to pinned buffer
        memcpy(h_in[s], h_input + offset, thisChunk * sizeof(float));

        // Async: H→D, then kernel, then D→H (all in same stream, ordered)
        cudaMemcpyAsync(d_in[s], h_in[s], thisChunk * sizeof(float),
                       cudaMemcpyHostToDevice, streams[s]);

        processKernel<<<grid, block, 0, streams[s]>>>(d_in[s], d_out[s], thisChunk);

        cudaMemcpyAsync(h_out[s], d_out[s], thisChunk * sizeof(float),
                       cudaMemcpyDeviceToHost, streams[s]);

        // If previous chunk in this stream is done, copy results to output
        if (chunk >= nStreams) {
            cudaStreamSynchronize(streams[s]);
            int prevOffset = (chunk - nStreams) * chunkSize;
            int prevChunk  = min(chunkSize, N - prevOffset);
            memcpy(h_output + prevOffset, h_out[s], prevChunk * sizeof(float));
        }
    }

    // Flush remaining chunks
    for (int s = 0; s < nStreams; s++) cudaStreamSynchronize(streams[s]);
    // Copy last outputs...

    // Cleanup
    cudaFreeHost(h_in[0]); cudaFreeHost(h_in[1]);
    cudaFreeHost(h_out[0]); cudaFreeHost(h_out[1]);
    cudaFree(d_in[0]); cudaFree(d_in[1]);
    cudaFree(d_out[0]); cudaFree(d_out[1]);
    cudaStreamDestroy(streams[0]); cudaStreamDestroy(streams[1]);
}

// Timeline with double-buffering (chunkSize = N/4):
//
// Stream 0: [H2D-0][KERNEL-0][D2H-0]
// Stream 1:        [H2D-1][KERNEL-1][D2H-1]
// Stream 0:               [H2D-2][KERNEL-2][D2H-2]
// Stream 1:                      [H2D-3][KERNEL-3][D2H-3]
//
// Total: ~= T_kernel (if compute-bound) or T_copy (if copy-bound)
// vs sequential: 4 * (T_H2D + T_kernel + T_D2H)

CUDA Events — Timing and Synchronization

// Timing with CUDA events (only way to accurately time GPU operations)
cudaEvent_t start, stop;
cudaEventCreate(&start);
cudaEventCreate(&stop);

cudaEventRecord(start, 0);             // record in stream 0 (default)
myKernel<<<grid, block>>>(d_A, d_B, N);
cudaEventRecord(stop, 0);

cudaEventSynchronize(stop);            // wait for stop event to complete
float ms;
cudaEventElapsedTime(&ms, start, stop);
printf("Kernel time: %.3f ms\n", ms);
printf("Bandwidth: %.2f GB/s\n",
       (2.0 * N * sizeof(float)) / (ms * 1e-3) / 1e9);

cudaEventDestroy(start);
cudaEventDestroy(stop);

// Cross-stream synchronization with events:
cudaEvent_t dep;
cudaEventCreate(&dep);
kernel1<<<grid, block, 0, stream1>>>(d_A, d_B, N);
cudaEventRecord(dep, stream1);          // dep fires when stream1 is here
cudaStreamWaitEvent(stream2, dep, 0);   // stream2 waits for dep before proceeding
kernel2<<<grid, block, 0, stream2>>>(d_B, d_C, N);  // uses d_B produced by kernel1

Stream Synchronization Methods

MethodWhat it waits forWho waits
cudaDeviceSynchronize()ALL operations on ALL streamsCPU blocks
cudaStreamSynchronize(s)All operations in stream sCPU blocks
cudaEventSynchronize(e)Event e to be recordedCPU blocks
cudaStreamWaitEvent(s, e)Event e before stream s continuesGPU stream waits, CPU continues
cudaStreamQuery(s)Non-blocking: poll if stream is doneCPU polls, no block

Concurrent Kernel Execution

Multiple kernels can run simultaneously on the GPU if:

  • They are in different (non-default) streams
  • The GPU has enough resources (SMs, registers, shared memory) for both
  • The GPU supports concurrent kernel execution (prop.concurrentKernels == 1)
// Two independent kernels running concurrently
cudaStream_t s1, s2;
cudaStreamCreate(&s1);
cudaStreamCreate(&s2);

// These launch almost simultaneously and run concurrently
smallKernel<<<16, 256, 0, s1>>>(d_A, N);   // uses 16 SMs
largeKernel<<<80, 256, 0, s2>>>(d_B, M);   // uses 80 SMs
// Total: 96 SMs used (A100 has 108), both run in parallel

cudaStreamSynchronize(s1);
cudaStreamSynchronize(s2);

cudaStreamDestroy(s1);
cudaStreamDestroy(s2);

Stream Priorities

// Query priority range (lower number = higher priority)
int leastPriority, greatestPriority;
cudaDeviceGetStreamPriorityRange(&leastPriority, &greatestPriority);
// typically: leastPriority=0, greatestPriority=-1

// Create high-priority stream for latency-sensitive work
cudaStream_t highPri, lowPri;
cudaStreamCreateWithPriority(&highPri, cudaStreamNonBlocking, greatestPriority);
cudaStreamCreateWithPriority(&lowPri,  cudaStreamNonBlocking, leastPriority);

// High-priority kernels preempt low-priority ones (compute preemption on Pascal+)
lowPriKernel<<<huge_grid, 256, 0, lowPri>>>(d_big_data, N);  // long background task
// ... some time later, urgent work arrives:
highPriKernel<<<small_grid, 256, 0, highPri>>>(d_urgent, M);  // preempts lowPri
