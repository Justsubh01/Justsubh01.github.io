Error Handling in CUDA — Never Trust cudaSuccess Blindly

CUDA's asynchronous execution model means errors surface far from their origin, often as a baffling wrong answer with no crash. This post arms you with the complete error-handling toolkit: the CUDA_CHECK macro, kernel error detection, the ten most dangerous error codes, compute-sanitizer, and a production-grade cuda_utils.h header you can drop into any project.

I have watched senior engineers spend three days debugging a GPU numerical error that a two-second run of compute-sanitizer would have diagnosed instantly. CUDA's error model is C-style — functions return error codes, and if you do not check them, failures are completely silent. Combine that with asynchronous kernel execution — where an error in kernel K can surface as an error code returned by cudaMemcpy called twenty lines later — and you have a system that demands disciplined error handling at every layer. This post gives you that discipline.

1. Why CUDA Errors Are Silent By Default

Every CUDA runtime API function returns a cudaError_t value — an integer enum. If the call succeeds, it returns cudaSuccess (0). If it fails, it returns a non-zero error code describing what went wrong. The critical detail: C does not require you to use a function's return value. The compiler issues no warning. Your program continues running. 

// This compiles cleanly with zero warnings:
cudaMalloc(&d_A, bytes);       // return value silently discarded
cudaMemcpy(d_A, h_A, bytes,
           cudaMemcpyHostToDevice);   // return value silently discarded
myKernel<<<grid, block>>>(d_A, d_B, N);
cudaMemcpy(h_C, d_C, bytes,
           cudaMemcpyDeviceToHost);   // return value silently discarded

Here is what a silent failure chain looks like at runtime: 

// Scenario: GPU is already at 99% memory usage.

// Call 1: cudaMalloc attempts to allocate 4 GB
cudaMalloc(&d_A, 4ULL * 1024 * 1024 * 1024);
// Returns: cudaErrorMemoryAllocation (2)
// Since we don't check: d_A is now NULL or an invalid address.

// Call 2: cudaMemcpy tries to copy to the invalid d_A
cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice);
// Returns: cudaErrorInvalidValue (1) because d_A == NULL
// Since we don't check: bytes are "copied" to address 0x0 (discarded).

// Call 3: Kernel runs with d_A = NULL
myKernel<<<4096, 256>>>(d_A, d_B, d_C, N);
// The kernel launch itself may succeed (it's just enqueueing a command).
// But when threads execute d_C[idx] = d_A[idx] + d_B[idx],
// they dereference NULL — an illegal memory access.

// Call 4: This cudaMemcpy finally fails visibly
cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost);
// Returns: cudaErrorIllegalAddress — the kernel had an illegal access.
// But we're now 4 calls away from the actual bug (the OOM on call 1).
// If we had checked call 1, we'd have exited immediately with a clear message.
A CUDA error from call N often surfaces only at call N+3, N+5, or later.
CUDA's asynchronous execution model means that kernel errors are "sticky" — they remain in the error state of the CUDA context until the next synchronizing call reads them. If you do not check every API call, you may spend hours staring at a failure that occurred dozens of lines before the crash site. The only defense is checking every single call.

Kernel launches are even more problematic. The triple-chevron syntax <<<...>>> returns no value at all: 

// __global__ functions always return void.
// The kernel launch syntax is special NVCC syntax that expands to a
// cudaLaunchKernel() call internally — but that return value is not
// surfaced to you at the call site.
//
// This is completely silent — no return value to check:
myKernel<<<grid, block>>>(args);

// Common mistakes that produce NO immediate error:
//  - blockDim.x > 1024 (exceeds hardware limit — caught by cudaGetLastError)
//  - kernel crashes with illegal memory access (caught by cudaDeviceSynchronize)
//  - kernel writes wrong data due to missing bounds check (silent forever)

The categories of what can go wrong, and when each surfaces:

Error Type When It Occurs How to Detect Detectability
API call failure (OOM, invalid arg) Immediately on the call Check return value of API call Easy with CUDA_CHECK
Launch config error (bad block size) At launch, before kernel runs cudaGetLastError() after launch Easy
Kernel execution error (illegal access) Async — at next sync call cudaDeviceSynchronize() or next cudaMemcpy Medium — error displaced from cause
Wrong computation (logic bug) Never crashes Result verification loop Hard — only if you verify output
Race condition (shared memory) Intermittent; input-dependent compute-sanitizer --tool racecheck Very hard without tooling

2. The CUDA_CHECK Macro — Full Anatomy

The industry-standard pattern for CUDA error handling is a macro that wraps every API call, checks the return value, prints a diagnostic message, and exits. Here is the canonical version with every design choice explained: 

// ─────────────────────────────────────────────────────────────────────
// CUDA_CHECK macro — production-ready version
// ─────────────────────────────────────────────────────────────────────
#define CUDA_CHECK(call)                                               \
  do {                                                                 \
    cudaError_t _err = (call);                                         \
    if (_err != cudaSuccess) {                                         \
      fprintf(stderr,                                                  \
              "CUDA error at %s:%d\n"                                  \
              "  Error code: %d (%s)\n"                                \
              "  Description: %s\n",                                   \
              __FILE__, __LINE__,                                       \
              (int)_err,                                               \
              cudaGetErrorName(_err),                                  \
              cudaGetErrorString(_err));                               \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
  } while (0)

// Design decisions explained:
//
// 1. do { ... } while(0) — the "do-while trick"
//    Without this, the macro expands to a bare block, which breaks in:
//    if (condition) CUDA_CHECK(cudaMalloc(...));
//    else doSomething();
//    The compiler sees: if (cond) do_stuff; ; else doSomething();
//    The extra semicolon after } creates an empty statement, and the else
//    has no matching if. With do-while(0), the macro behaves exactly like
//    a single statement — you can put a semicolon after it anywhere.
//
// 2. cudaError_t _err = (call)
//    The underscore prefix on _err prevents collision with any local variable
//    named 'err' in the caller's scope. Macro hygiene.
//
// 3. __FILE__ and __LINE__
//    Built-in preprocessor macros that expand to the source file name and
//    line number at the CALL SITE. This is what makes the error message useful:
//    "CUDA error at vectorAdd.cu:47" instead of "error at cuda_utils.h:15".
//
// 4. cudaGetErrorName() vs cudaGetErrorString()
//    cudaGetErrorName()   returns the enum name: "cudaErrorMemoryAllocation"
//    cudaGetErrorString() returns a human sentence: "out of memory"
//    Include both — the name is grep-able in documentation, the string is readable.
//
// 5. exit(EXIT_FAILURE) vs throw / return
//    In C programs: exit() is appropriate for fatal errors.
//    In C++ programs: you may prefer throw std::runtime_error(msg) for
//    exception-based cleanup. The exit() approach leaks host memory but is
//    simpler — the OS reclaims it anyway.

Here is a complete vectorAdd.cu rewritten with CUDA_CHECK on every single CUDA call — this is what production code looks like: 

#include <cuda_runtime.h>
#include <stdio.h>
#include <stdlib.h>
#include <math.h>

#define CUDA_CHECK(call)                                               \
  do {                                                                 \
    cudaError_t _err = (call);                                         \
    if (_err != cudaSuccess) {                                         \
      fprintf(stderr, "CUDA error at %s:%d — [%s] %s\n",            \
              __FILE__, __LINE__,                                       \
              cudaGetErrorName(_err),                                  \
              cudaGetErrorString(_err));                               \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
  } while (0)

__global__ void vectorAdd(const float* __restrict__ a,
                          const float* __restrict__ b,
                          float* __restrict__ c, int n)
{
    int idx = blockIdx.x * blockDim.x + threadIdx.x;
    if (idx < n) c[idx] = a[idx] + b[idx];
}

int main(void)
{
    const int    N     = 1 << 20;
    const size_t bytes = (size_t)N * sizeof(float);

    float *h_A = (float*)malloc(bytes);
    float *h_B = (float*)malloc(bytes);
    float *h_C = (float*)malloc(bytes);
    if (!h_A || !h_B || !h_C) { perror("malloc"); exit(EXIT_FAILURE); }

    for (int i = 0; i < N; i++) { h_A[i] = (float)i; h_B[i] = (float)(N - i); }

    float *d_A, *d_B, *d_C;
    CUDA_CHECK(cudaMalloc((void**)&d_A, bytes));   // line 34
    CUDA_CHECK(cudaMalloc((void**)&d_B, bytes));   // line 35
    CUDA_CHECK(cudaMalloc((void**)&d_C, bytes));   // line 36

    CUDA_CHECK(cudaMemcpy(d_A, h_A, bytes, cudaMemcpyHostToDevice));  // line 38
    CUDA_CHECK(cudaMemcpy(d_B, h_B, bytes, cudaMemcpyHostToDevice));  // line 39

    const int blockSize = 256;
    const int gridSize  = (N + blockSize - 1) / blockSize;

    cudaEvent_t evStart, evStop;
    CUDA_CHECK(cudaEventCreate(&evStart));
    CUDA_CHECK(cudaEventCreate(&evStop));
    CUDA_CHECK(cudaEventRecord(evStart));

    vectorAdd<<<gridSize, blockSize>>>(d_A, d_B, d_C, N);

    // Catch launch-configuration errors (synchronous — returns immediately):
    CUDA_CHECK(cudaGetLastError());                // line 51

    CUDA_CHECK(cudaEventRecord(evStop));
    // Catch kernel execution errors (asynchronous — need to sync first):
    CUDA_CHECK(cudaEventSynchronize(evStop));      // line 54: GPU finishes here
    // After EventSynchronize, any async kernel error becomes visible:
    CUDA_CHECK(cudaGetLastError());                // line 56

    float ms = 0.0f;
    CUDA_CHECK(cudaEventElapsedTime(&ms, evStart, evStop));
    printf("Time: %.3f ms\n", ms);

    CUDA_CHECK(cudaEventDestroy(evStart));
    CUDA_CHECK(cudaEventDestroy(evStop));

    CUDA_CHECK(cudaMemcpy(h_C, d_C, bytes, cudaMemcpyDeviceToHost));  // line 63

    int errors = 0;
    for (int i = 0; i < N; i++)
        if (fabsf(h_C[i] - (float)N) > 1e-3f) errors++;
    printf("%s (%d errors)\n", errors ? "FAILED" : "PASSED", errors);

    CUDA_CHECK(cudaFree(d_A));
    CUDA_CHECK(cudaFree(d_B));
    CUDA_CHECK(cudaFree(d_C));
    free(h_A); free(h_B); free(h_C);
    return 0;
}

When an error is caught, the output looks like this:

# Simulated: cudaMalloc fails because GPU is out of memory CUDA error at vectorAdd.cu:34 — [cudaErrorMemoryAllocation] out of memory # Simulated: wrong blockSize exceeds hardware limit (blockSize=2048) CUDA error at vectorAdd.cu:51 — [cudaErrorInvalidValue] invalid argument # Simulated: kernel illegal memory access (missing bounds check) CUDA error at vectorAdd.cu:56 — [cudaErrorIllegalAddress] an illegal memory access was encountered

cudaGetLastError vs cudaPeekAtLastError
These two functions are often confused. The difference matters in error-handling loops: 

// cudaGetLastError(): returns the last error AND resets it to cudaSuccess.
// Use this when you want to consume and clear the error.
cudaError_t err = cudaGetLastError();  // reads and clears
// Next call to cudaGetLastError() will return cudaSuccess (if nothing new failed).

// cudaPeekAtLastError(): returns the last error WITHOUT clearing it.
// Use this when you want to inspect without consuming.
cudaError_t err = cudaPeekAtLastError();  // reads only
// The error is still set — the next cudaGetLastError() returns the same value.

// Practical implication:
// If you call cudaGetLastError() BEFORE clearing a sticky error,
// the function reports a stale error from a much earlier call.
// Always call cudaGetLastError() right after the operation you want to check,
// not later. If you want to clear a known stale error before a fresh check:
cudaGetLastError();   // discard any stale error
myKernel<<<g, b>>>(args);
CUDA_CHECK(cudaGetLastError());  // now a clean check for this launch only

3. Kernel Errors: Synchronous vs Asynchronous Detection

Kernels introduce a two-stage error detection problem. There are two distinct things that can go wrong: the launch can fail, and the execution can fail. Each is detected differently. 

// STAGE 1: Launch-time errors (synchronous — detected immediately)
//
// These are caught by cudaGetLastError() right after the launch syntax.
// The kernel has not started executing yet.
//
// Common causes:
//   - blockDim.x * blockDim.y * blockDim.z > 1024 (too many threads per block)
//   - Passing NULL as a required parameter (device function pointers)
//   - Insufficient shared memory requested (> hardware maximum, typically 48–163 KB)
//   - Too many registers per thread (compiler can calculate this at compile time)

myKernel<<<gridSize, 2048>>>(args);  // 2048 > 1024 limit!
cudaError_t launchErr = cudaGetLastError();
// launchErr == cudaErrorInvalidValue  — caught immediately, before any GPU work

// STAGE 2: Execution-time errors (asynchronous — detected at sync point)
//
// The kernel launched successfully (correct config) but crashed during execution.
// The GPU records the error internally. It surfaces at the NEXT synchronizing call.
//
// Common causes:
//   - Out-of-bounds memory access (idx >= N without bounds check)
//   - Null pointer dereference inside kernel
//   - Stack overflow (recursive __device__ functions going too deep)
//   - NaN propagation causing hardware exceptions (on some configs)

myKernel<<<gridSize, blockSize>>>(d_ptr_that_is_null, d_B, d_C, N);
cudaError_t launchErr = cudaGetLastError();
// launchErr == cudaSuccess  (launch config was fine!)

cudaError_t execErr = cudaDeviceSynchronize();
// execErr == cudaErrorIllegalAddress  — caught here, NOT at the launch line

The combined CUDA_CHECK_KERNEL macro handles both stages: 

// Combined kernel error check macro.
// Use this after EVERY kernel launch in debug/development builds.
// In production, consider removing the cudaDeviceSynchronize() for performance
// (it serializes CPU and GPU) and only keeping cudaGetLastError().
#define CUDA_CHECK_KERNEL()                                            \
  do {                                                                 \
    /* Stage 1: catch launch-config errors (synchronous) */           \
    cudaError_t _e1 = cudaGetLastError();                              \
    if (_e1 != cudaSuccess) {                                          \
      fprintf(stderr,                                                  \
              "CUDA kernel LAUNCH error at %s:%d\n"                    \
              "  [%s] %s\n",                                           \
              __FILE__, __LINE__,                                       \
              cudaGetErrorName(_e1),                                   \
              cudaGetErrorString(_e1));                                \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
    /* Stage 2: catch execution errors (asynchronous) */              \
    cudaError_t _e2 = cudaDeviceSynchronize();                         \
    if (_e2 != cudaSuccess) {                                          \
      fprintf(stderr,                                                  \
              "CUDA kernel EXEC error at %s:%d\n"                      \
              "  [%s] %s\n",                                           \
              __FILE__, __LINE__,                                       \
              cudaGetErrorName(_e2),                                   \
              cudaGetErrorString(_e2));                                \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
  } while (0)

// Usage:
vectorAdd<<<gridSize, blockSize>>>(d_A, d_B, d_C, N);
CUDA_CHECK_KERNEL();   // catches both launch and execution errors

Summary of which errors are synchronous vs asynchronous:

Error Scenario Type Detection Call Notes
blockDim > 1024 Synchronous cudaGetLastError() Caught before GPU starts
gridDim > 2^31-1 Synchronous cudaGetLastError() Hardware launch limit
Shared memory > limit Synchronous cudaGetLastError() Usually 48KB or 96KB
Illegal memory access Asynchronous cudaDeviceSynchronize() Most common runtime crash
Stack overflow (recursive) Asynchronous cudaDeviceSynchronize() Device stack is limited
Assertion failure (__assert) Asynchronous cudaDeviceSynchronize() CUDA assert in device code
Wrong result (logic bug) Never crashes Result verification No CUDA mechanism detects this

4. The Ten Most Dangerous CUDA Error Codes

There are over 100 CUDA error codes, but 80% of real bugs produce one of these ten. Recognizing them instantly saves hours of debugging.

Error Code Integer Value Typical Cause Immediate Fix
cudaErrorInvalidValue 1 NULL pointer argument, zero size, out-of-range enum value. Extremely common: cudaMalloc(&d_A, 0) or wrong cudaMemcpyKind. Check all arguments to the failing call. Print them before the call.
cudaErrorMemoryAllocation 2 GPU out of memory. cudaMalloc requested more bytes than available. Check nvidia-smi for current usage. Reduce batch size; free unused allocations; check for leaks (not calling cudaFree).
cudaErrorInitializationError 3 CUDA driver not initialized. Calling CUDA API before any CUDA function was called, or in a forked process. Call cudaSetDevice(0) first. Never use CUDA across fork() without reinitializing.
cudaErrorNoDevice 100 No CUDA-capable GPU found. Driver not installed, GPU not present, or inside a container with no GPU passthrough. Run nvidia-smi. Check docker run --gpus. Verify GPU is visible with cudaGetDeviceCount().
cudaErrorInvalidDevice 101 cudaSetDevice(N) called with N >= number of GPUs. Multi-GPU code with wrong device index. Call cudaGetDeviceCount() and validate index before cudaSetDevice().
cudaErrorInvalidDevicePointer 17 Passing a CPU (host) pointer where a GPU pointer is expected in cudaMemcpy or a kernel argument. Classic h_ vs d_ confusion. Enforce h_ / d_ naming convention. Use cudaPointerGetAttributes() to verify pointer type.
cudaErrorIllegalAddress 700 Kernel accessed unmapped memory: idx >= N without bounds check, NULL pointer dereference inside kernel, or double-freed device memory. Add bounds check. Run compute-sanitizer --tool memcheck for exact thread/address. This is the most common serious error.
cudaErrorLaunchOutOfResources 7 Kernel requires more registers or shared memory than the SM can provide. Usually caused by high register usage per thread combined with large blockDim. Reduce blockDim; add __launch_bounds__(maxThreads, minBlocks); check register usage with nvcc --ptxas-options=-v.
cudaErrorInvalidResourceHandle 33 Using a destroyed or uninitialized cudaEvent_t, cudaStream_t, or cudaGraph_t. Often caused by calling cudaEventDestroy before cudaEventSynchronize. Ensure create-before-use and destroy-after-use ordering. Check for double-destroy.
cudaErrorNotReady 600 cudaEventQuery() or cudaStreamQuery() returns this when the event/stream has not completed yet. NOT a fatal error — it is a polling status code. This is expected in polling loops. Call cudaEventSynchronize() to wait instead of busy-polling.
cudaErrorIllegalAddress (error 700) is the sneakiest one.
It is an asynchronous error, so it appears at the next sync call after the kernel crash. The error message says nothing about which thread or which array caused it. compute-sanitizer --tool memcheck gives you the exact threadIdx, blockIdx, and the bad address. Never debug error 700 without compute-sanitizer — it would take you orders of magnitude longer manually.

5. compute-sanitizer — Valgrind for CUDA

compute-sanitizer is the official NVIDIA dynamic analysis tool. It instruments GPU code at runtime to catch errors that are otherwise silent or near-impossible to locate manually. Think of it as Valgrind, AddressSanitizer, and ThreadSanitizer combined — but for GPUs. 

# compile with device debug symbols for best line-level output
nvcc -g -G -O0 -o vecadd vecadd.cu

# Run with memcheck (default tool — catches illegal memory accesses)
compute-sanitizer --tool memcheck ./vecadd

# Run with racecheck (catches shared memory race conditions)
compute-sanitizer --tool racecheck ./vecadd

# Run with synccheck (catches missing or misplaced __syncthreads())
compute-sanitizer --tool synccheck ./vecadd

# Run with initcheck (catches reading uninitialized device memory)
compute-sanitizer --tool initcheck ./vecadd

# Run ALL tools in sequence (slowest but most thorough)
compute-sanitizer --tool memcheck \
  compute-sanitizer --tool racecheck \
  ./vecadd

# Useful flags:
#   --show-backtrace=yes  : print host call stack at kernel launch time
#   --error-exitcode=1    : exit with non-zero on any detected error (for CI)
#   --log-file report.txt : write full report to file
compute-sanitizer --tool memcheck --show-backtrace=yes --error-exitcode=1 ./vecadd

Here is what compute-sanitizer output looks like for a kernel with a missing bounds check (idx not limited to < N):

========= COMPUTE-SANITIZER ========= Error: process didn't terminate successfully ========= Invalid __global__ read of size 4 bytes ========= at 0x00000068 in vectorAdd(float const*, float const*, float*, int) ========= by thread (192, 0, 0) in block (3906, 0, 0) ========= Address 0x7f33c4000200 is out of bounds ========= Device Frame: vectorAdd(float const*, float const*, float*, int) ========= Saved host backtrace up to driver entry point at kernel launch time ========= Host Frame: libcuda.so.1 (cuLaunchKernel + 0x1b0) ========= Host Frame: ./vecadd (cudaLaunchKernel + 0x3a) ========= Host Frame: ./vecadd (main + 0x1a4) [vectorAdd.cu:52] ========= ========= cudaDeviceSynchronize returned cudaErrorIllegalAddress ========= (cudaError_t 700) ========= ========= 1 error

That output tells you precisely: thread 192 in block 3906 executed an out-of-bounds read of 4 bytes. Block 3906 * 256 + 192 = 999,744 + 192 = 999,936. If N = 1,000,003 and blockSize = 256, gridSize = 3907. The last valid thread is at index 1,000,002. Thread 3906*256+192 = 999,936 is actually valid. But if we had N=999,000 with the same launch config, threads 999,000..999,191 in block 3902 would be illegal. The exact numbers pinpoint the bug.

What each tool catches:

Tool What it catches Performance overhead Recommended use
memcheck Out-of-bounds reads/writes, use-after-free, misaligned access, uninitialized writes 10–50x slowdown Every commit, every CI run
racecheck Shared memory data races (concurrent read/write without __syncthreads) 20–200x slowdown When adding shared memory optimization
synccheck Divergent __syncthreads() (some threads reach barrier, others don't) 5–20x slowdown Kernels with conditional __syncthreads()
initcheck Reading device memory before any write (uninitialized reads) 20–100x slowdown During development; catches missing cudaMemset
Run compute-sanitizer in every CI test.
Add compute-sanitizer --tool memcheck --error-exitcode=1 ./your_test to your CI pipeline. It catches 80% of CUDA memory bugs automatically in seconds. The performance overhead does not matter in CI (tests run on pre-production data). Engineers who skip this step spend weeks finding bugs that compute-sanitizer catches in two seconds. This is not an exaggeration — it is a routine observation in GPU teams.

6. Seven-Step GPU Debugging Workflow

When you encounter a GPU bug that compute-sanitizer doesn't immediately diagnose (logic errors, wrong numerical results, intermittent crashes), follow this workflow:

Step 1: Reproduce with tiny N and print everything.

// Reduce N to 8 or 16. Print every input and output.
// If the bug disappears at N=8, it's a bounds/alignment issue.
// If the bug persists at N=8, you can inspect every single element.
const int N = 8;
// ... run kernel ...
for (int i = 0; i < N; i++) {
    printf("[%d] A=%.3f B=%.3f C_gpu=%.3f C_expected=%.3f %s\n",
           i, h_A[i], h_B[i], h_C[i], h_A[i]+h_B[i],
           (fabsf(h_C[i] - h_A[i] - h_B[i]) < 1e-3f) ? "OK" : "BAD");
}

Step 2: Run compute-sanitizer with memcheck.

compute-sanitizer --tool memcheck --show-backtrace=yes ./your_program
# If it reports errors: fix them first before investigating further.

Step 3: Add printf inside the kernel.

// YES, CUDA kernels can call printf(). This is the single most powerful
// debugging technique for GPU code. The output is buffered in a 1 MB
// circular FIFO on the device and flushed after kernel completion.
// Each printf() call is SLOW (serialized through a mutex-like mechanism),
// so only print for small N or specific threads.

__global__ void vectorAdd(const float* a, const float* b, float* c, int n)
{
    int idx = blockIdx.x * blockDim.x + threadIdx.x;

    // Print only for the first 8 threads to avoid flooding output
    if (idx < 8) {
        printf("[GPU] block=%d thread=%d idx=%d  a[idx]=%.3f b[idx]=%.3f\n",
               blockIdx.x, threadIdx.x, idx, a[idx], b[idx]);
    }

    if (idx < n)
        c[idx] = a[idx] + b[idx];

    // Print result for first 8
    if (idx < 8) {
        printf("[GPU] idx=%d  c[idx]=%.3f (expected %.3f)\n",
               idx, c[idx], a[idx] + b[idx]);
    }
}
# Output from the GPU printf (note: ordering may vary between blocks/threads): [GPU] block=0 thread=0 idx=0 a[idx]=0.000 b[idx]=1048576.000 [GPU] block=0 thread=1 idx=1 a[idx]=1.000 b[idx]=1048575.000 [GPU] block=0 thread=2 idx=2 a[idx]=2.000 b[idx]=1048574.000 [GPU] block=0 thread=0 idx=0 c[idx]=1048576.000 (expected 1048576.000) [GPU] block=0 thread=1 idx=1 c[idx]=1048576.000 (expected 1048576.000) [GPU] block=0 thread=2 idx=2 c[idx]=1048576.000 (expected 1048576.000)

Step 4: Increase printf buffer size for large-N debugging.

// The default printf FIFO is 1 MB. For N=10000 with verbose output,
// this fills up and later messages are silently dropped.
// Increase it before any kernel launches:
size_t fifo_size = 64 * 1024 * 1024;  // 64 MB
CUDA_CHECK(cudaDeviceSetLimit(cudaLimitPrintfFifoSize, fifo_size));
// Call this at the beginning of main(), before any kernel launches.
// It takes effect for all subsequent kernels in that process.

Step 5: Reduce to a minimal reproducer.

// A minimal reproducer is the smallest possible program that shows the bug.
// Remove ALL unrelated code. This has two benefits:
//   1. The bug becomes obvious when isolated.
//   2. You can share it for help without leaking proprietary code.
//
// Template for a minimal CUDA reproducer:

#include <cuda_runtime.h>
#include <stdio.h>

__global__ void buggyKernel(float *d, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    // REPRODUCE THE BUG HERE with minimal code
    if (i < n) d[i] = 1.0f / (float)(i - 500);  // example: division issue
}

int main() {
    float *d; cudaMalloc(&d, 1000 * sizeof(float));
    buggyKernel<<<4, 256>>>(d, 1000);
    cudaDeviceSynchronize();
    printf("Error: %s\n", cudaGetErrorString(cudaGetLastError()));
    cudaFree(d);
}

Step 6: Use cuda-gdb for interactive debugging.

# Compile with debug symbols (mandatory for cuda-gdb)
nvcc -g -G -O0 -o vecadd vecadd.cu

# Launch cuda-gdb
cuda-gdb ./vecadd

# Common cuda-gdb commands:
(cuda-gdb) run                     # start the program

7. Production Template: cuda_utils.h

Here is a complete, drop-in header file you can add to any CUDA project. It provides error-checking macros for CUDA runtime, cuBLAS, cuDNN, and NCCL; safe allocation helpers; and a logging integration pattern. 

// ═══════════════════════════════════════════════════════════════════
// cuda_utils.h — Production CUDA error handling header
// Drop this into any CUDA project. Include it everywhere.
// ═══════════════════════════════════════════════════════════════════
#pragma once

#include <cuda_runtime.h>
#include <stdio.h>
#include <stdlib.h>

// ─────────────────────────────────────────────────────────────────
// Core CUDA runtime error check
// ─────────────────────────────────────────────────────────────────
#define CUDA_CHECK(call)                                               \
  do {                                                                 \
    cudaError_t _err = (call);                                         \
    if (_err != cudaSuccess) {                                         \
      fprintf(stderr,                                                  \
              "\n[CUDA ERROR] %s:%d\n"                                 \
              "  Error:       %s (%d)\n"                               \
              "  Description: %s\n\n",                                 \
              __FILE__, __LINE__,                                       \
              cudaGetErrorName(_err), (int)_err,                       \
              cudaGetErrorString(_err));                               \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
  } while (0)

// ─────────────────────────────────────────────────────────────────
// Kernel launch error check (two-stage: launch + execution)
// Use this after EVERY kernel launch in development.
// In production: keep cudaGetLastError(), remove cudaDeviceSynchronize()
// ─────────────────────────────────────────────────────────────────
#define CUDA_CHECK_KERNEL()                                            \
  do {                                                                 \
    cudaError_t _e1 = cudaGetLastError();                              \
    if (_e1 != cudaSuccess) {                                          \
      fprintf(stderr,                                                  \
              "\n[CUDA KERNEL LAUNCH ERROR] %s:%d\n"                   \
              "  Error: %s (%d) — %s\n\n",                           \
              __FILE__, __LINE__,                                       \
              cudaGetErrorName(_e1), (int)_e1,                         \
              cudaGetErrorString(_e1));                                \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
    cudaError_t _e2 = cudaDeviceSynchronize();                         \
    if (_e2 != cudaSuccess) {                                          \
      fprintf(stderr,                                                  \
              "\n[CUDA KERNEL EXEC ERROR] %s:%d\n"                     \
              "  Error: %s (%d) — %s\n\n",                           \
              __FILE__, __LINE__,                                       \
              cudaGetErrorName(_e2), (int)_e2,                         \
              cudaGetErrorString(_e2));                                \
      exit(EXIT_FAILURE);                                              \
    }                                                                  \
  } while (0)

// ─────────────────────────────────────────────────────────────────
// cuBLAS error check (if using cuBLAS)
// ─────────────────────────────────────────────────────────────────

// ─────────────────────────────────────────────────────────────────
// Safe allocation helpers
// ─────────────────────────────────────────────────────────────────

// Allocate N floats on device. Aborts on failure.
static inline float* cuda_malloc_float(int n) {
    float* ptr = NULL;
    CUDA_CHECK(cudaMalloc((void**)&ptr, (size_t)n * sizeof(float)));
    return ptr;
}

// Allocate N bytes of pinned (page-locked) host memory.
// Pinned memory enables async transfers and higher PCIe throughput.
static inline void* cuda_malloc_pinned(size_t bytes) {
    void* ptr = NULL;
    CUDA_CHECK(cudaMallocHost(&ptr, bytes));
    return ptr;
}

// Upload: host → device (synchronous)
static inline void cuda_upload(void* d_dst, const void* h_src, size_t bytes) {
    CUDA_CHECK(cudaMemcpy(d_dst, h_src, bytes, cudaMemcpyHostToDevice));
}

// Download: device → host (synchronous)
static inline void cuda_download(void* h_dst, const void* d_src, size_t bytes) {
    CUDA_CHECK(cudaMemcpy(h_dst, d_src, bytes, cudaMemcpyDeviceToHost));
}

// Zero-initialize device memory (like memset for GPU)
static inline void cuda_zero(void* d_ptr, size_t bytes) {
    CUDA_CHECK(cudaMemset(d_ptr, 0, bytes));
}

// Print GPU memory usage (useful for debugging OOM errors)
static inline void cuda_print_memory_usage(const char* label) {
    size_t free_bytes, total_bytes;
    CUDA_CHECK(cudaMemGetInfo(&free_bytes, &total_bytes));
    size_t used_bytes = total_bytes - free_bytes;
    fprintf(stderr, "[GPU Memory] %s: %.1f MB used / %.1f MB total (%.1f MB free)\n",
            label,
            used_bytes  / (1024.0 * 1024.0),
            total_bytes / (1024.0 * 1024.0),
            free_bytes  / (1024.0 * 1024.0));
}

// ─────────────────────────────────────────────────────────────────
// CUDA event timer helper
// ─────────────────────────────────────────────────────────────────
typedef struct {
    cudaEvent_t start;
    cudaEvent_t stop;
} CudaTimer;

static inline CudaTimer cuda_timer_create(void) {
    CudaTimer t;
    CUDA_CHECK(cudaEventCreate(&t.start));
    CUDA_CHECK(cudaEventCreate(&t.stop));
    return t;
}

static inline void cuda_timer_start(CudaTimer* t) {
    CUDA_CHECK(cudaEventRecord(t->start));
}

// Returns elapsed milliseconds; also synchronizes the GPU.
static inline float cuda_timer_stop(CudaTimer* t) {
    CUDA_CHECK(cudaEventRecord(t->stop));
    CUDA_CHECK(cudaEventSynchronize(t->stop));
    float ms = 0.0f;
    CUDA_CHECK(cudaEventElapsedTime(&ms, t->start, t->stop));
    return ms;
}

static inline void cuda_timer_destroy(CudaTimer* t) {
    CUDA_CHECK(cudaEventDestroy(t->start));
    CUDA_CHECK(cudaEventDestroy(t->stop));
}

// ─────────────────────────────────────────────────────────────────
// Device info printer (call at startup to log GPU capabilities)
// ─────────────────────────────────────────────────────────────────
static inline void cuda_print_device_info(int device) {
    cudaDeviceProp prop;
    CUDA_CHECK(cudaGetDeviceProperties(&prop, device));
    fprintf(stderr,
            "[CUDA Device %d] %s\n"
            "  Compute capability : %d.%d\n"
            "  Global memory      : %.1f GB\n"
            "  Memory bandwidth   : %.1f GB/s\n"
            "  SM count           : %d\n"
            "  Max threads/block  : %d\n"
            "  Warp size          : %d\n"
            "  L2 cache size      : %d MB\n",
            device, prop.name,
            prop.major, prop.minor,
            prop.totalGlobalMem / (1024.0 * 1024.0 * 1024.0),
            2.0 * prop.memoryClockRate * (prop.memoryBusWidth / 8) / 1.0e6,
            prop.multiProcessorCount,
            prop.maxThreadsPerBlock,
            prop.warpSize,
            prop.l2CacheSize / (1024 * 1024));
}

#include "cuda_utils.h"

int main(void) {
    // Log GPU info at startup
    cuda_print_device_info(0);
    cuda_print_memory_usage("startup");

    const int N = 1 << 20;
    const size_t bytes = (size_t)N * sizeof(float);

    // Allocate
    float *d_A = cuda_malloc_float(N);
    float *d_B = cuda_malloc_float(N);
    float *d_C = cuda_malloc_float(N);
    cuda_print_memory_usage("after alloc");

    // Initialize host data (omitted for brevity)
    // Upload
    cuda_upload(d_A, h_A, bytes);
    cuda_upload(d_B, h_B, bytes);

    // Time the kernel
    CudaTimer timer = cuda_timer_create();
    cuda_timer_start(&timer);

    vectorAdd<<<(N+255)/256, 256>>>(d_A, d_B, d_C, N);
    CUDA_CHECK_KERNEL();

    float ms = cuda_timer_stop(&timer);
    cuda_timer_destroy(&timer);

    double bw = (3.0 * N * sizeof(float)) / (ms * 1e-3) / 1e9;
    printf("Time: %.3f ms   BW: %.1f GB/s\n", ms, bw);

    // Download and verify (omitted for brevity)
    // Free
    CUDA_CHECK(cudaFree(d_A));
    CUDA_CHECK(cudaFree(d_B));
    CUDA_CHECK(cudaFree(d_C));
    return 0;
}

Development vs Production error handling trade-offs.
In development: check everything, call CUDA_CHECK_KERNEL() after every launch (the cudaDeviceSynchronize() inside will hurt throughput but catch bugs instantly). In production: keep CUDA_CHECK on allocations (OOM is fatal and common), replace CUDA_CHECK_KERNEL() with just CUDA_CHECK(cudaGetLastError()) (catches launch errors without forcing sync), and add full error checks only at pipeline boundaries. The cuda_utils.h above can be configured with a preprocessor flag: #ifdef DEBUG_CUDA around the synchronize call in CUDA_CHECK_KERNEL.

The discipline of CUDA error handling is ultimately simple: check every return value, use compute-sanitizer in CI, and understand the asynchronous error displacement model. Engineers who internalize these three rules debug GPU code in minutes rather than days. The production header above encodes years of practical experience into patterns you can apply immediately.
