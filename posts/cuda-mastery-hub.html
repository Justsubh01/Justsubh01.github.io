CUDA Mastery Series — Complete Learning Path

A structured, bottom-up journey through CUDA GPU programming — from C fundamentals and CPU/GPU architecture all the way to warp-level optimization, profiling, and NVIDIA interview preparation. 16 in-depth posts, hundreds of code examples, and everything you need to go from zero to production-ready CUDA.

This series covers everything I studied while going deep on GPU programming. Each post stands alone but they're designed to be read in order — concepts build on each other. Phase 1 establishes foundations, Phases 2–4 are core CUDA, Phase 5 is performance engineering, and Phase 6 is interview prep.

Total reading: ~6 hours across all posts. Total coding: implement everything yourself as you go — that's where the learning actually happens.

Quick Reference

  Essential formulas you must know:
  ─────────────────────────────────────────────────────────────
  Global thread index (1D):  idx = blockIdx.x * blockDim.x + threadIdx.x
  Grid size (ceiling div):   gridSize = (N + blockSize - 1) / blockSize
  Arithmetic Intensity:      AI = FLOPs / bytes_from_DRAM
  Occupancy:                 active_warps / max_warps_per_SM
  Warp size:                 32 (always)
  Max threads per block:     1024
  Shared memory per SM:      48–164 KB (depends on config)
  Registers per SM:          65536 × 32-bit

  Memory latency reference:
    Registers:      ~1 cycle
    Shared memory:  ~5 cycles
    L2 cache:       ~200 cycles
    Global memory:  ~800 cycles
    Host memory:    millions of cycles (PCIe)

  CUDA_CHECK macro (mandatory):
    #define CUDA_CHECK(call) do { \
        cudaError_t e = (call); \
        if (e != cudaSuccess) { \
            fprintf(stderr, "CUDA error %s:%d — %s\n", \
                    __FILE__, __LINE__, cudaGetErrorString(e)); \
            exit(EXIT_FAILURE); \
        } \
    } while(0)

Prerequisites

  • Basic Linux command line
  • Access to a CUDA-capable GPU (GTX 1060+ or any data center GPU)
  • CUDA toolkit installed (nvcc --version should work)
