Warps, SIMT, and Warp Divergence — The Execution Model Under the Hood

Every CUDA performance conversation eventually comes back to warps. A warp is 32 threads executing the same instruction in lockstep. Understanding how warps are scheduled, what happens when they diverge, and how the SM hides memory latency through warp switching is essential for writing fast kernels.

Threads are the CUDA abstraction. Warps are the CUDA reality. The hardware never actually runs individual threads — it runs warps of 32 threads in lockstep. Every performance characteristic of a CUDA kernel — occupancy, latency hiding, divergence cost, register pressure — is ultimately a warp-level story.

Without divergence (left): all 32 threads execute the same path simultaneously — full SIMT throughput. With if/else divergence (right): the warp serializes both paths, masking inactive threads on each pass. For equal-probability branches, effective throughput is halved.

What Is a Warp

A warp is a group of 32 consecutive threads scheduled and executed together. When you launch a block of 256 threads, the hardware automatically divides them into 8 warps of 32: threads 0–31 in warp 0, threads 32–63 in warp 1, etc.

Key warp facts:

  • Warp size is always 32 — this has been true since CUDA 1.0 and is architecturally fixed
  • Blocks are always a multiple of 32 threads in effective scheduling (a block of 33 threads has 2 warps, with 31 active in the second warp)
  • The lane ID of a thread within its warp is threadIdx.x % 32
  • The warp ID within a block is threadIdx.x / 32
  • All 32 threads in a warp issue the same instruction each cycle (SIMT)
__global__ void warpInfo() {
    int tid    = threadIdx.x;
    int laneId = tid % 32;        // position within warp (0-31)
    int warpId = tid / 32;        // which warp within block
    int globalWarpId = (blockIdx.x * blockDim.x + tid) / 32;

    printf("thread %3d: lane=%2d, warp_in_block=%d, global_warp=%d\n",
           tid, laneId, warpId, globalWarpId);
}

SIMT Execution Model

SIMT stands for Single Instruction, Multiple Threads. All 32 threads in a warp execute the same instruction at the same time, but each operates on different data (different register values, different memory addresses computed from their thread index).

This is similar to SIMD (Single Instruction Multiple Data, as in x86 AVX), but with a key difference: SIMT threads can diverge (take different code paths through if/else). SIMD cannot — all SIMD lanes must execute the same operation.

Warp Scheduling — Latency Hiding

The SM's 4 warp schedulers interleave warps every cycle. When warp A issues a memory load and stalls for ~800 cycles, the scheduler immediately switches to warp B, C, D... — keeping the execution units busy. By the time the scheduler returns to warp A, the data has (ideally) arrived.

An A100 SM has 4 independent warp schedulers, each capable of issuing one instruction per cycle. Each SM can hold up to 64 warps (2048 threads) simultaneously. When a warp stalls (waiting for a memory load, a function call, or a sync), the scheduler immediately switches to another ready warp — zero idle cycles.

This is the GPU's primary latency-hiding mechanism. Global memory latency is ~800 cycles. If you have 20 warps active per SM, and each warp issues a load every 40 instructions, you have 20 × 40 = 800 instructions of other work to execute while each warp waits. The GPU never stalls — it just keeps switching.

The warp scheduler timeline:
Cycle 1: Warp 0 issues load (800-cycle stall starts)
Cycle 2: Warp 1 executes arithmetic
Cycle 3: Warp 2 executes arithmetic
...
Cycle 800: Warp 0's data arrives, scheduler can issue next instruction
→ Zero cycles wasted if enough warps are active

Occupancy — How Many Warps Are Active

Occupancy = (active warps on SM) / (maximum warps on SM). For an A100: max warps = 64, so occupancy = active_warps / 64.

Three resources limit occupancy:

  1. Registers: 65,536 total per SM ÷ registers_per_thread ÷ 32 = max warps
  2. Shared memory: total_smem / smem_per_block = max blocks (each block has at least 1 warp)
  3. Block size: max_threads_per_SM / threads_per_block = max blocks

Warp Divergence in Depth

Warp divergence occurs when threads within the same warp take different code paths through an if/else, switch, or loop with variable iteration count.

How the hardware handles it:

  1. Warp reaches branch: evaluate predicate for all 32 threads
  2. 32-bit mask created: 1 = thread takes branch, 0 = thread skips
  3. Execute "if" body with threads where mask=1, others are masked off (no-op)
  4. Execute "else" body with threads where mask=0, others are masked off
  5. Warp reconverges at merge point
// Example: 50% divergence
__global__ void divergentKernel(float* out, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i >= n) return;

    // Threads 0-15 in each warp take if, threads 16-31 take else
    // Both paths execute sequentially with half the threads masked
    if (i % 2 == 0) {
        out[i] = sqrtf(out[i]);     // heavy: only even threads active
    } else {
        out[i] = out[i] * 2.0f;    // light: only odd threads active
    }
    // Reconverge here
}

// Effective throughput: ~50% of non-divergent version

The performance cost depends on the balance between branches. Worst case: 50/50 split (half the warp idles during each branch). Best case: one branch is essentially a no-op (the masked threads do nothing meaningful).

Bounds-check divergence is fine. The most common divergence pattern — if (i >= N) return; — only affects the last warp of a grid when N is not a multiple of 32. This is at most 1 warp out of thousands and has negligible impact. Don't over-optimize this.

Eliminating Divergence

Three strategies:

1. Sort data so threads in same warp do the same thing

// DIVERGENT: sparse operation, each thread may or may not work
__global__ void sparseNaive(float* data, int* mask, float* out, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i >= n) return;
    if (mask[i]) {          // some threads skip, some don't → divergence
        out[i] = sqrtf(data[i]);
    }
}

// BETTER: pre-sort active indices, all threads always work
// (requires prefix sum to compact active indices)
__global__ void sparseCompact(float* data, int* activeIdx, float* out, int nActive) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i >= nActive) return;
    int idx = activeIdx[i];     // all threads take same path, no divergence
    out[idx] = sqrtf(data[idx]);
}

2. Hoist branches out of loops

// SLOW: branch inside loop re-evaluated every iteration
__global__ void branchInLoop(float* A, float* B, float* C, int n, bool useA) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    for (int j = 0; j < n; j++) {
        if (useA) C[i] += A[j];   // branch evaluated n times
        else      C[i] += B[j];
    }
}

// FAST: branch hoisted, separate kernels, no divergence
__global__ void useAKernel(float* A, float* C, int n) { ... }
__global__ void useBKernel(float* B, float* C, int n) { ... }
// Choose kernel on CPU, zero divergence in either

3. Use arithmetic instead of branches

// DIVERGENT: if/else inside hot loop
__global__ void withBranch(float* out, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i >= n) return;
    if (out[i] < 0.0f) out[i] = 0.0f;  // ReLU with branch
}

// NO DIVERGENCE: arithmetic equivalent (compiler often does this automatically)
__global__ void noBranch(float* out, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    if (i >= n) return;
    out[i] = fmaxf(out[i], 0.0f);  // no branch, single instruction
}

Warp-Level Primitives

Since Volta (compute capability 7.0), CUDA provides direct warp-level intrinsics that let threads within a warp communicate without shared memory. These are faster than shared memory for intra-warp communication. 

// __shfl_sync: broadcast a value from one lane to all others
__global__ void broadcastFromLane0(float* out, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    float val = (threadIdx.x % 32 == 0) ? out[i] : 0.0f;  // only lane 0 has data
    // Broadcast lane 0's value to all other lanes in the warp
    val = __shfl_sync(0xFFFFFFFF, val, 0);  // srcLane = 0
    out[i] = val;
}

// __shfl_down_sync: shift values down (for reduction)
__device__ float warpReduceSum(float val) {
    unsigned mask = 0xFFFFFFFF;  // all 32 lanes participate
    for (int offset = 16; offset > 0; offset /= 2) {
        val += __shfl_down_sync(mask, val, offset);
    }
    return val;  // lane 0 has the sum of all 32 lanes
}

// __ballot_sync: which threads have predicate true?
__global__ void countActive(int* data, int* count, int n) {
    int i = blockIdx.x * blockDim.x + threadIdx.x;
    bool active = (i < n) && (data[i] > 0);
    // 32-bit mask: bit k = 1 if thread k in this warp has active=true
    unsigned mask = __ballot_sync(0xFFFFFFFF, active);
    if (threadIdx.x % 32 == 0) {
        atomicAdd(count, __popc(mask));  // popcount = number of active threads
    }
}

Measuring Divergence with ncu

ncu --metrics   smsp__thread_inst_executed_pred_on_per_inst_executed.ratio,  smsp__average_warp_latency_per_inst_issued.ratio   ./myprogram
