Dynamic Programming: From Memoization to Tabulation

DP is the topic that separates L4 from L5 at Google. It's not about memorizing solutions — it's about recognizing overlapping subproblems and defining the right state. Master the framework: identify state, write recurrence, implement top-down then convert to bottom-up. 15 problems from Fibonacci to LCS to knapsack.

Dynamic programming is intimidating until you realize it's just recursion with caching. Every DP problem has the same structure: define a state, write a recursive solution, add memoization, optionally convert to iterative. The hard part is step one — defining the right state. Get that right and the rest follows.

When to Use DP: The Two Conditions

  DP applies when a problem has BOTH:

  1. Optimal Substructure:
     The optimal solution to the problem contains optimal solutions
     to subproblems.
     "The longest path from A to C passing through B uses the
     longest path from A to B and longest path from B to C."

  2. Overlapping Subproblems:
     The same subproblems are solved multiple times.
     Without caching, we waste exponential time.

  Contrast with:
  - Greedy: optimal substructure but no overlapping subproblems
    (each step makes a locally optimal choice, never revisits)
  - Divide and conquer: overlapping? Usually not (merge sort subproblems don't repeat)

  Trigger words in problem statement:
  ─────────────────────────────────────────────────────────────
  "number of ways to..."              → counting DP
  "minimum/maximum cost to..."        → optimization DP
  "can you achieve/reach..."          → boolean DP
  "longest/shortest subsequence..."   → sequence DP
  "how many distinct..."              → counting DP

The DP Framework: 4 Steps

  Step 1: Define the state
    dp[i] = "the answer to the subproblem for the first i elements"
    What does dp[i] represent? This is 80% of the problem.

  Step 2: Write the recurrence
    How does dp[i] depend on dp[j] for j < i?
    This is the "transition function."

  Step 3: Identify base cases
    What are the smallest subproblems with known answers?
    Usually dp[0] and dp[1].

  Step 4: Determine computation order
    Which order lets you compute dp[i] from previously computed values?
    Usually left-to-right for 1D, row-by-row for 2D.

  Example: Fibonacci
  ─────────────────────────────────────────────────────────────
  Step 1: dp[i] = i-th Fibonacci number
  Step 2: dp[i] = dp[i-1] + dp[i-2]   (recurrence)
  Step 3: dp[0] = 0, dp[1] = 1         (base cases)
  Step 4: left to right                 (i depends on i-1, i-2)

Top-Down (Memoization) vs Bottom-Up (Tabulation)

  Top-Down: Start from the target, recurse down, cache results.
  ─────────────────────────────────────────────────────────────
  from functools import lru_cache

  @lru_cache(maxsize=None)
  def fib(n):
      if n <= 1: return n
      return fib(n-1) + fib(n-2)
  # Natural, mirrors recursive thinking
  # Only computes states that are actually needed
  # O(n) time, O(n) space (call stack + cache)

  Without lru_cache:
  def fib(n, memo={}):
      if n in memo: return memo[n]
      if n <= 1: return n
      memo[n] = fib(n-1) + memo + fib(n-2, memo)
      return memo[n]

  Bottom-Up: Start from base cases, build up to target.
  ─────────────────────────────────────────────────────────────
  def fib(n):
      if n <= 1: return n
      dp = [0] * (n + 1)
      dp[1] = 1
      for i in range(2, n+1):
          dp[i] = dp[i-1] + dp[i-2]
      return dp[n]
  # No recursion → no stack overflow risk
  # Often faster (no function call overhead)
  # Sometimes allows space optimization

  Space-optimized (Fibonacci only needs last 2):
  def fib(n):
      if n <= 1: return n
      a, b = 0, 1
      for _ in range(2, n+1):
          a, b = b, a + b
      return b
  # O(1) space — rolling array technique

  Which to use in interviews?
  ─────────────────────────────────────────────────────────────
  Start with top-down (memoization) — it's easier to write correctly.
  Convert to bottom-up if asked for space optimization.
  Space-optimize the bottom-up table if possible (interview bonus).

1D DP Problems

Climbing Stairs

  # How many ways to climb n stairs (1 or 2 steps at a time)?
  def climb_stairs(n):
      if n <= 2: return n
      a, b = 1, 2
      for _ in range(3, n+1):
          a, b = b, a + b
      return b
  # dp[i] = ways to reach step i = dp[i-1] + dp[i-2]
  # This is Fibonacci!

House Robber

  # Rob houses in a row, can't rob adjacent houses.
  def rob(nums):
      if len(nums) == 1: return nums[0]
      prev2, prev1 = 0, 0
      for num in nums:
          curr = max(prev1, prev2 + num)
          prev2, prev1 = prev1, curr
      return prev1
  # dp[i] = max money robbing houses 0..i
  # dp[i] = max(dp[i-1], dp[i-2] + nums[i])

  # House Robber II (circular: first and last adjacent):
  def rob_circular(nums):
      def rob_linear(houses):
          prev2 = prev1 = 0
          for h in houses:
              prev2, prev1 = prev1, max(prev1, prev2 + h)
          return prev1
      return max(nums[0], rob_linear(nums[1:]), rob_linear(nums[:-1]))

Coin Change (Minimum Coins)

  def coin_change(coins, amount):
      dp = [float('inf')] * (amount + 1)
      dp[0] = 0  # 0 coins needed to make amount 0
      for a in range(1, amount + 1):
          for coin in coins:
              if coin <= a:
                  dp[a] = min(dp[a], dp[a - coin] + 1)
      return dp[amount] if dp[amount] != float('inf') else -1
  # dp[a] = min coins to make amount a
  # Transition: for each coin, dp[a] = min(dp[a], dp[a-coin]+1)
  # O(amount * len(coins)) time, O(amount) space

Coin Change II (Count Ways)

  def change(amount, coins):
      dp = [0] * (amount + 1)
      dp[0] = 1  # one way to make 0: use no coins
      for coin in coins:          # process one coin type at a time
          for a in range(coin, amount + 1):
              dp[a] += dp[a - coin]
      return dp[amount]
  # Key: iterate coins outer, amounts inner → avoids counting [1,2] and [2,1] as different
  # This is the "unbounded knapsack" pattern

Word Break

  def word_break(s, wordDict):
      word_set = set(wordDict)
      n = len(s)
      dp = [False] * (n + 1)
      dp[0] = True  # empty string can be segmented
      for i in range(1, n + 1):
          for j in range(i):
              if dp[j] and s[j:i] in word_set:
                  dp[i] = True
                  break
      return dp[n]
  # dp[i] = can s[0:i] be segmented?
  # O(n^2 * L) where L = max word length for set lookup

2D DP Problems

Unique Paths

  # Robot moving from top-left to bottom-right, only right or down.
  def unique_paths(m, n):
      dp = [[1] * n for _ in range(m)]
      for r in range(1, m):
          for c in range(1, n):
              dp[r][c] = dp[r-1][c] + dp[r][c-1]
      return dp[m-1][n-1]
  # Space optimization: only need previous row
  def unique_paths_opt(m, n):
      dp = [1] * n
      for _ in range(1, m):
          for c in range(1, n):
              dp[c] += dp[c-1]
      return dp[n-1]

Longest Common Subsequence (LCS)

  def lcs(text1, text2):
      m, n = len(text1), len(text2)
      dp = [[0] * (n+1) for _ in range(m+1)]
      for i in range(1, m+1):
          for j in range(1, n+1):
              if text1[i-1] == text2[j-1]:
                  dp[i][j] = dp[i-1][j-1] + 1
              else:
                  dp[i][j] = max(dp[i-1][j], dp[i][j-1])
      return dp[m][n]

  # State: dp[i][j] = LCS length of text1[0..i] and text2[0..j]
  # Recurrence:
  #   if chars match: dp[i][j] = dp[i-1][j-1] + 1
  #   else: max(skip from text1, skip from text2)
  # O(m*n) time, O(m*n) space (or O(n) with rolling array)

Edit Distance (Levenshtein)

  def min_distance(word1, word2):
      m, n = len(word1), len(word2)
      dp = list(range(n+1))  # base case: transform empty to word2[0..j]
      for i in range(1, m+1):
          prev = dp[0]
          dp[0] = i  # base case: transform word1[0..i] to empty
          for j in range(1, n+1):
              temp = dp[j]
              if word1[i-1] == word2[j-1]:
                  dp[j] = prev      # no operation needed
              else:
                  dp[j] = 1 + min(prev,   # replace
                                  dp[j],   # delete from word1
                                  dp[j-1]) # insert into word1
              prev = temp
      return dp[n]
  # dp[i][j] = min edits to convert word1[0..i] to word2[0..j]

0/1 Knapsack

  def knapsack(weights, values, capacity):
      n = len(weights)
      dp = [0] * (capacity + 1)
      for i in range(n):
          # Iterate backwards to avoid using item i multiple times
          for w in range(capacity, weights[i]-1, -1):
              dp[w] = max(dp[w], dp[w - weights[i]] + values[i])
      return dp[capacity]
  # dp[w] = max value with capacity w
  # Backwards iteration ensures each item used at most once
  # O(n * capacity) time, O(capacity) space

  # Subset Sum (special case: values = weights, target = capacity):
  def can_sum(nums, target):
      dp = [False] * (target + 1)
      dp[0] = True
      for num in nums:
          for t in range(target, num-1, -1):
              dp[t] = dp[t] or dp[t-num]
      return dp[target]

Interval DP

  # For problems where we build solutions from sub-intervals.
  # Pattern: dp[i][j] = answer for interval [i, j]
  # Solve smaller intervals first, build up to [0, n-1]

  ─── Burst Balloons ──────────────────────────────────────
  def max_coins(nums):
      nums = [1] + nums + [1]  # add boundary balloons
      n = len(nums)
      dp = [[0] * n for _ in range(n)]

      for length in range(2, n):       # interval length
          for left in range(0, n - length):
              right = left + length
              for k in range(left+1, right):  # k is the LAST balloon burst in [left, right]
                  dp[left][right] = max(dp[left][right],
                      dp[left][k] + nums[left]*nums[k]*nums[right] + dp[k][right])
      return dp[0][n-1]

  ─── Palindrome Partitioning II (Min Cuts) ───────────────
  def min_cut(s):
      n = len(s)
      # is_palindrome[i][j] = True if s[i:j+1] is palindrome
      is_pal = [[False]*n for _ in range(n)]
      for i in range(n-1, -1, -1):
          for j in range(i, n):
              is_pal[i][j] = (s[i]==s[j]) and (j-i<=2 or is_pal[i+1][j-1])

      # dp[i] = min cuts for s[0:i+1]
      dp = list(range(-1, n))  # dp[0] = -1 (no cut needed for s[0:0])
      for i in range(n):
          for j in range(i+1):
              if is_pal[j][i]:
                  dp[i+1] = min(dp[i+1], dp[j] + 1)
      return dp[n]

DP on Sequences

Longest Increasing Subsequence (LIS)

  # O(n^2) DP:
  def lis(nums):
      n = len(nums)
      dp = [1] * n  # dp[i] = LIS ending at index i
      for i in range(1, n):
          for j in range(i):
              if nums[j] < nums[i]:
                  dp[i] = max(dp[i], dp[j] + 1)
      return max(dp)

  # O(n log n) with patience sorting:
  import bisect
  def lis_fast(nums):
      tails = []  # tails[i] = smallest tail of IS of length i+1
      for num in nums:
          pos = bisect.bisect_left(tails, num)
          if pos == len(tails): tails.append(num)
          else: tails[pos] = num  # replace to maintain smallest possible tail
      return len(tails)

Maximum Product Subarray

  def max_product(nums):
      max_p = min_p = result = nums[0]
      for num in nums[1:]:
          # Negatives flip max and min!
          candidates = (num, max_p * num, min_p * num)
          max_p = max(candidates)
          min_p = min(candidates)
          result = max(result, max_p)
      return result
  # Track both max AND min (negative * negative = positive)

15 DP Problems Ranked by Importance

  Must-know (appear repeatedly in interviews):
  ───────────────────────────────────────────────────────────
  1.  Fibonacci / Climbing Stairs           (1D, O(n))
  2.  House Robber                          (1D, O(n))
  3.  Coin Change (min coins)               (1D knapsack, O(n*k))
  4.  Longest Increasing Subsequence        (O(n^2) or O(n log n))
  5.  Longest Common Subsequence            (2D, O(m*n))
  6.  Edit Distance                         (2D, O(m*n))
  7.  0/1 Knapsack / Subset Sum             (bounded knapsack)
  8.  Unique Paths                          (2D grid DP)
  9.  Maximum Subarray (Kadane's)           (1D, O(n))
  10. Word Break                            (1D, O(n^2))

  Important (appear in senior interviews):
  ───────────────────────────────────────────────────────────
  11. Coin Change II (count ways)           (unbounded knapsack)
  12. Palindromic Substrings / LPS          (2D palindrome DP)
  13. Burst Balloons                        (interval DP)
  14. Longest Palindromic Subsequence       (2D, O(n^2))
  15. Partition Equal Subset Sum            (boolean knapsack)

  State design patterns:
  ───────────────────────────────────────────────────────────
  dp[i]       → answer considering first i elements
  dp[i][j]    → answer for subproblem on [i, j] or (i, j) state
  dp[i][w]    → knapsack: first i items, capacity w
  dp[i][k]    → first i elements using at most k operations
  dp[s]       → bitmask DP: set s of items included (n ≤ 20)

Bitmask DP (for Small n ≤ 20)

  # When n is small, represent subsets as bitmasks.
  # dp[mask] = answer when the set of items = mask is selected.

  # Travelling Salesman Problem:
  def tsp(dist, n):
      full_mask = (1 << n) - 1  # all cities visited
      # dp[mask][i] = min cost to visit cities in mask, ending at city i
      dp = [[float('inf')] * n for _ in range(1 << n)]
      dp[1][0] = 0  # start at city 0, only city 0 visited

      for mask in range(1 << n):
          for last in range(n):
              if not (mask >> last & 1): continue  # last not in mask
              if dp[mask][last] == float('inf'): continue
              for nxt in range(n):
                  if mask >> nxt & 1: continue  # nxt already visited
                  new_mask = mask | (1 << nxt)
                  dp[new_mask][nxt] = min(dp[new_mask][nxt],
                                         dp[mask][last] + dist[last][nxt])
      # Return to start: min dp[full_mask][i] + dist[i][0]
      return min(dp[full_mask][i] + dist[i][0] for i in range(1, n))
